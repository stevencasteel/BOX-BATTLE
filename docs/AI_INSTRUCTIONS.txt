1. TERMINAL EXECUTION & HEREDOC PROTOCOLS (CRITICAL)
----------------------------------------------------
To prevent Zsh parsing failures, environment errors, variable corruption, or script truncation on macOS systems, strictly adhere to the following execution protocols:

1.1 Quoted Starting Delimiters (Shell Protection)
    - Always wrap the starting heredoc delimiter in single quotes (e.g., cat << 'EOF' > path/to/file or cat << 'ENDOFSCRIPT').
    - Unquoted delimiters allow the terminal shell (Zsh/Bash) to perform immediate variable expansion ($variable) and backtick execution (`` ` ``) *before* writing the text to the file. This process strips or corrupts raw JavaScript, TypeScript, and CSS rules.

1.2 Zero Shell Comments
    - Never include hash symbols (#) inside any active terminal or shell code blocks.
    - Copy-pasting shell commands containing hash lines directly into macOS interactive Zsh terminals triggers parsing errors or partial executions. Keep shell blocks strictly functional and describe operations in the accompanying markdown text instead.

1.3 Delimiter Syntax and Formatting
    - The terminating delimiter (e.g., EOF or ENDOFSCRIPT) must sit alone on its own line with zero leading or trailing whitespace.
    - If any space, tab, or comment character is placed on the delimiter line, the shell will fail to recognize the boundary and keep appending input, causing truncation or terminal lockup.

1.4 Temporary Script ESM Standard
    - Since this project operates with "type": "module", all temporary Node.js scripts must use ES Module syntax (e.g., use "import fs from 'fs';" instead of "const fs = require('fs');").

1.5 Script Execution and Automatic Cleanup
    - All file patches using temporary Node.js scripts must always append the self-delete command on the exact same execution line (e.g., "node ./patch.js && rm ./patch.js") to prevent cluttering the repository.

1.6 Zsh Compatibility & Special Characters
    - Do not use unescaped exclamation marks (!) inside shell string variables. Zsh treats them as history expansion operators and will fail to execute the command. 

1.7 Surgical Patches vs. Overwrites
    - For small-to-medium files, perform full overwrites (cat << 'EOF' > file).
    - For large files, write a temporary Node.js script using fs.readFileSync and regex/string replacement (as defined in 1.4 and 1.5) to avoid file truncation during large output generation.
